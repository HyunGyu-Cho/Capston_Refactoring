# refactoring-docs Index

## 이 폴더를 보는 방법
이 문서는 "어떤 정보를 어디서 찾는지" 안내하는 지도입니다.

## 폴더 구조
- `01-product/requirements`: 서비스 요구사항, 비기능 요구사항
- `02-architecture`: 모듈 구조, 공통 규칙, API 계약
- `03-domain/<module>`: 모듈별 설계/ERD/이슈/계획
- `04-data`: DB 운영 전략
- `05-operations`: 운영/개발 회고 문서
- `99-archive/legacy`: 이전 구조 백업

## 빠른 시작 경로
1. 먼저 `01-product/requirements/REQUIREMENTS.md`
2. 다음 `02-architecture/module-map/MODULE_SPECIFIC.md`
3. 공통 규칙은 `02-architecture/common/COMMON_RULE.md`
4. 구현 전에는 `03-domain/<module>/` 문서 확인
